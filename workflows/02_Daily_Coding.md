# Daily Coding Workflow

## Phase 1 - Start Session

- [ ] Open the project in VS Code.
- [ ] Open the terminal.
- [ ] Activate the virtual environment.
- [ ] Pull latest changes from GitHub.
- [ ] Review `TODO.md`.
- [ ] Review `REQUIREMENTS.md`.
- [ ] Choose one clear goal for the session.

---

## Phase 2 - Study Warmup

- [ ] Complete due Anki reviews.
- [ ] Review yesterday's notes if needed.
- [ ] Complete one short drill if available.

---

## Phase 3 - Development

- [ ] Work on the chosen goal.
- [ ] Make small changes.
- [ ] Run the program frequently.
- [ ] Read error messages carefully.
- [ ] Update `TODO.md` as tasks change.

---

## Phase 4 - Review Changes

- [ ] Run the program.
- [ ] Confirm today's goal was completed or partially completed.
- [ ] Run `git status`.
- [ ] Run `git diff`.
- [ ] Review changed files.
- [ ] Remove unnecessary debug code.

---

## Phase 5 - Commit and Push

- [ ] Stage the correct files.
- [ ] Run `git diff --staged`.
- [ ] Commit one logical change.
- [ ] Push changes to GitHub.

---

## Phase 6 - End Session

- [ ] Update `CHANGELOG.md` if meaningful progress was made.
- [ ] Update handbook notes if a new concept was learned.
- [ ] Add flashcards if something needs memorization.
- [ ] Write down the next task for tomorrow.
