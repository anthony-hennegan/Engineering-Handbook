# Python Drill 015 - Function Design

## Objective

Practice designing small functions with clear responsibilities.

---

## Rules

- No handbook
- No flashcards
- No internet
- Run the program from the terminal
- Write the code from memory
- Use the exercise file only after your first attempt

---

## Tasks

- [ ] Create a new Python file for the drill.
- [ ] Create a variable named `reader_name`.
- [ ] Store a messy name string in `reader_name`.
- [ ] Define a function named `format_name`.
- [ ] Give `format_name` one parameter named `name`.
- [ ] Inside `format_name`, clean the name with `.strip().title()`.
- [ ] Return the cleaned name.
- [ ] Define a function named `greet_reader`.
- [ ] Give `greet_reader` one parameter named `name`.
- [ ] Inside `greet_reader`, print a greeting using `name`.
- [ ] Call `format_name` and store the returned value back into `reader_name`.
- [ ] Call `greet_reader` and pass in `reader_name`.
- [ ] Create a list named `books`.
- [ ] Add at least three book dictionaries to the list.
- [ ] Each book dictionary should have:
  - `title`
  - `author`
  - `checked_out`
- [ ] Define a function named `display_books`.
- [ ] Give `display_books` one parameter named `book_list`.
- [ ] Inside `display_books`, loop through `book_list`.
- [ ] Print each book title and author.
- [ ] Define a function named `is_available`.
- [ ] Give `is_available` one parameter named `book`.
- [ ] Inside `is_available`, return the opposite of the book’s `checked_out` value.
- [ ] Loop through `books`.
- [ ] For each book, call `is_available`.
- [ ] Print whether each book is available or checked out.
- [ ] Add a comment explaining why `format_name` returns a value.
- [ ] Add a comment explaining why `greet_reader` prints instead of returns.
- [ ] Add a comment explaining why `is_available` returns a Boolean.
- [ ] Add a comment explaining what “one function one job” means.
